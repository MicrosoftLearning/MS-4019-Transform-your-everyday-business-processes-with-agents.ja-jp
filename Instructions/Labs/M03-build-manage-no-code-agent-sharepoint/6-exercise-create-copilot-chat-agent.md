この演習では、Microsoft 365 Copilot Chat 用の独自のカスタム エージェントを作成します。 カスタム エージェントは、組織内の特定のワークフロー、タスク、またはドメインを支援するように設計された調整済みの AI アシスタントです。 また、仕事に関連しないプライベートなプロジェクトにも役立ちます。 エージェントを設計することを通じて、役割、指示、データ ソース、アクセス許可などの構成オプションを使って Copilot の動作と応答の指針を示す方法を調査する必要があります。 この演習の目標は、Microsoft 365 Copilot Chat 内で作成するエージェントの目的と機能について批判的に考える方法を理解することです。

### 演習

この演習では、創造性を発揮して、自分にとって関心の高いエージェントを設計することをお勧めします。 たとえば、現実のビジネス課題を解決したり、会社の特定の分野で生産性を向上させたり、個人的に関心のあるテーマに取り組んだりすることが考えられます。 それぞれの受講者が独自のパーソナル エージェントを作成するので、以下の手順では、各フィールドに入力する実際の値を指定するのではなく、エージェントを構成する方法の構造と順序に焦点を当てます。 エージェントを作成する際に、各フィールドに入力する内容を自分で決める必要があります。

1. **Microsoft Edge** で新しいタブを開き、次の URL を入力します: [**https://M365copilot.com**](https://M365copilot.com)
1. **Microsoft 365** のナビゲーション ウィンドウで **[Copilot]** を選択します。
1. **Copilot Studio** ウィンドウには、既定で **[説明]** タブが表示されています。 既存のテンプレートに基づいたエージェントにする場合は、表示されているテンプレートのどれかを選択するか、**[すべてのテンプレートの表示]** を選択します。 **[すべてのテンプレートの表示]** を選択した場合は、テンプレートの一覧をスクロールし、任意のテンプレートを選択します。 テンプレートを選択したら、選択したテンプレートからエージェントに割り当てられたスターター プロンプトに注意してください。
1. ページ左側のテンプレートの下に表示される **[メッセージを入力してください]** フィールドは、エージェントを構成するのに使用します。 ページ右側のスターター プロンプトの下に表示される **[メッセージを入力してください]** フィールドは、エージェントをテストするのに使用します。 ここではエージェントを構成するので、残りの手順では、ページの左側に表示される **[メッセージを入力してください]** フィールドを使用します。 このフィールドに、このエージェントが何を行うように設計されているかを説明する短い概要を記述し、**[送信]** (右向きの矢印) アイコンを選択します。
1. 今入力した説明でエージェントの名前を指定しなかった場合、通常は Copilot Studio によって提案された名前が提供され、その名前の確認または新しい名前の入力を求められます。 それに応じて答える必要があります。 
1. この時点で、通常は Copilot Studio でエージェントに関する指示を調整するように求められます。 これは、一連の質問によって行われる場合があります。 エージェントの説明が満足のいくものになるまで、各質問に適宜答えます。 また、一般公開されている Web サイトをこのエージェントのデータ ソースとして追加するかどうか、Copilot Studio で確認を求められる場合もあります。 ここで応答することもできますが、通常は、以降の手順にある **[構成]** タブでナレッジ ソースを割り当てます。 
1. エージェントの説明が完成したら、フォームの上部にある **[構成]** タブを選択します。 
1. **[構成]** タブで、**[名前]**、**[説明]**、**[指示]** の各フィールドを確認します。 必要な変更があれば加えます。 指示は、入力した **[説明]** と、**[説明]** タブで答えたフォローアップの質問に対する回答に対応するように、Copilot Studio によって作成されたものであることに注意してください。指示は本質的に、入力した説明よりもかなり詳細になる傾向があることに留意してください。 エージェントの動作とユーザーへの応答の指針を示すうえで、指示が自分の意図と合致していることを確認します。 必要に応じて変更を加えます。
1. **[ナレッジ]** セクションでは、エージェントが参照する必要がある関連するデータ ソース (SharePoint サイト、ドキュメント、サード パーティの統合など) を追加する必要があります。

    > [!WARNING]
    > 公開 Web サイトや業務データを追加しなくても、エージェントは要求に応答できる可能性がありますが、一般的な知識の質問への回答、基本的な推論、一般化されたテキストの生成など、基になる大規模言語モデル (LLM) の一般的な機能のみを使用することになります。 そのため、特定の業務データまたは公開 Web サイトをエージェントの基盤とすることをお勧めします。そうしないと、エージェントの応答が、ビジネス固有のタスクに必要なコンテキストや関連性を欠いたものになる可能性があります。

1. **[機能]** セクションでは、必要に応じて、**[コード インタープリターター]** と **[画像ジェネレーター]** のトグル スイッチを選択してツールを有効にすることができます。 どちらのツールも既定では無効になっています。
1. **[スターター プロンプト]** セクションには、前出の **[説明]** タブでエージェントの説明を入力したときに Copilot Studio で自動的に作成された、6 つのスターター プロンプトが表示されます。既存のプロンプトを編集または削除することも、新しいプロンプトを追加することもできます。
1. 必要に応じて、ページ右側のスターター プロンプトの下にあるプロンプト フィールドにメッセージを入力して、エージェントをテストできます。 必要なだけエージェントをテストし、意図したとおりに動作または応答していない場合は、構成を調整できます。 
1. エージェントが正しく構成されたと思われたら、ページの上部にある **[作成]** ボタンを選択します。  
1. エージェントが作成されると、**[エージェントが正常に作成されました]** ウィンドウが表示されます。 共有設定を変更する場合は、**[共有設定の変更]** リンクを選択します。 そうすると、ユーザー アクセス設定を構成して、組織内でだれがこのエージェントを使用できるかを定義できます。 既定では、自分だけがエージェントを使用できます。 共有設定を構成したら (または自分専用にそのままにして)、**[エージェントに移動する]** ボタンを選択してエージェントを開きます。 
1. エージェントのウィンドウで、ナビゲーション ウィンドウにエージェントがどのように表示されるかを確認します。 スターター プロンプトのどれかを入力するか、他のカスタム プロンプトを入力して、エージェントをテストします。 
1. エージェントを使い終わったら、ナビゲーション ウィンドウで **[すべてのエージェント]** を選択します。 表示された **[エージェント ストア]** ウィンドウで、**[自分のエージェント]** セクションにエージェントがどのように表示されているかを確認します。 このエージェントにはエージェント ストアからアクセスできます。または、ペインのエージェントの一覧の下にエージェントを常に表示する場合は、ナビゲーション ウィンドウでエージェントの横に表示されるピン留めアイコンを選択できます。
